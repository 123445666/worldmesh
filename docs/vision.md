# Vision

## Short version

WorldMesh is GitHub for real-world coordination.

Not code collaboration, but collaboration on public problems, shared resources, field work, logistics, and community action.

## The problem

Human coordination is fragmented across too many systems:

- spreadsheets
- WhatsApp groups
- email chains
- local documents
- task trackers
- mapping tools
- funding reports

That fragmentation creates operational blindness.

People cannot easily answer:

- What exactly is the problem?
- Who is already working on it?
- What is blocked?
- What resources are missing?
- What depends on what?
- What progress has actually been made?

## The vision

Create an open-source platform where groups can:

- define problems clearly
- structure projects underneath them
- break work into tasks
- record decisions and rationale
- link resources and constraints
- coordinate across geography
- publish progress transparently

## What it should not become

WorldMesh should **not** become:

- a surveillance platform
- a centralized command system
- a social credit mechanism
- a prestige dashboard for institutions
- a vague AI planner with no accountability

## What success looks like

A local mutual aid network, civic group, or response team can use WorldMesh to understand:

- what needs doing
- who can do it
- what is blocked
- what resources are needed
- what changed this week

## Strategic wedge

Start with one narrow but meaningful use case:

**community and nonprofit coordination for place-based work**

That is concrete enough to design for and broad enough to grow from.
