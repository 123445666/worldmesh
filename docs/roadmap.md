# Roadmap

## Phase 0 — Concept

- define vision
- define domain model
- define MVP scope
- choose licensing strategy
- create repo scaffold

## Phase 1 — MVP

### Goals

- create problems, projects, tasks
- assign owners and collaborators
- track dependencies and blockers
- attach locations
- attach resources
- maintain decision logs
- simple public project view

### Deliverables

- web app scaffold
- database schema
- CRUD for core entities
- basic authentication
- role-based permissions
- activity feed
- map view for location-linked work

## Phase 2 — Coordination features

- notifications
- timeline/milestone views
- dependency graph visualization
- organization workspaces
- imports from CSV/spreadsheets
- evidence attachments

## Phase 3 — Federation and governance

- federation model
- inter-instance discovery
- signed event streams
- governance templates
- moderation and safety controls

## Phase 4 — Ecosystem

- public APIs
- SDK
- plugins/integrations
- analytics dashboards
- reporting exports

## Out of scope for MVP

- predictive AI decision-making
- automated authority over resource allocation
- biometric identity
- advanced financial disbursement
- global universal ontology of all social problems

## First recommended pilot

A city-scale volunteer and mutual aid coordination deployment.
