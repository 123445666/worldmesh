# WorldMesh

WorldMesh is an open-source coordination platform for real-world problems.

Instead of trying to "manage the world," it helps communities, nonprofits, mutual-aid groups, responders, and civic teams coordinate work in a transparent way.

## Core idea

Most tools handle only one slice of coordination:

- task managers handle tasks
- maps handle places
- CRMs handle contacts
- grant systems handle funding
- chat tools handle conversation

WorldMesh aims to connect the full chain:

**problem -> project -> task -> people -> resources -> action -> evidence**

## Principles

- **Open by default** where appropriate
- **Local autonomy** over central control
- **Transparent decisions** with visible audit trails
- **Federation-friendly architecture**
- **Permissioned privacy** for sensitive work
- **Real-world usefulness over techno-utopian fluff**

## Example use cases

- disaster relief coordination
- mutual aid networks
- local infrastructure repair
- food bank logistics
- housing repair queues
- volunteer mobilization
- NGO field operations

## MVP

The first version should do a few things well:

1. Create a **problem**
2. Create **projects** under that problem
3. Break projects into **tasks**
4. Link tasks to **people**, **organizations**, and **resources**
5. Attach **locations**
6. Track **status**, blockers, and dependencies
7. Publish a simple public-facing progress view

## Why this matters

Real-world work often fails because coordination is fragmented.
Not because people do not care, but because:

- nobody sees the whole system
- dependencies are hidden
- effort gets duplicated
- bottlenecks are invisible
- decisions are not documented

WorldMesh is meant to make collaboration legible.
